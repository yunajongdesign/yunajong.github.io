    Phase One: UI Style V.S. Game Style


    We tested two distinct styles—one based on UI design, the other on in-game visual style—to identify the more effective visual direction.

    Phase Two: M Symbol V.S. Buy

     

    We refined the chosen UI style by testing different information display formats. User behavior data guided the decision, helping determine the most effective UI presentation for engagement and clarity.

    Optimizing Game UX Through Data Analysis

    Case Two: Auto Play Usage

     

    By analyzing data from 11 games' autoplay usage, we assessed user behavior and completion patterns to determine whether additional autoplay count options were necessary, ensuring that design decisions were driven by actual player engagement.

    Optimizing Game UX Through Interview

    Case Three: Trigger Bonus Feature Sounds

     

    Conducted user interviews to select the most suitable sound for the Trigger Bonus Feature. Participants were first briefed on the brand identity and sound expectations, then asked to blind-test different sound options. Final audio was refined based on their feedback to ensure brand alignment and player impact.
